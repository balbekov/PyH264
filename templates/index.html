Encoding Pipeline

1Load
Load Parse input image or Y4M video file. Convert to grayscale (Y channel) and pad dimensions to multiples of 16 for macroblock alignment.
2Partition
Partition Split the frame into a hierarchy: Slices (rows of 16px) → MacroBlocks (16×16 pixels) → TransformBlocks (4×4 pixels). Each frame contains 16 transform blocks per macroblock.
3Predict
Intra Prediction Predict pixel values using neighboring blocks. Modes: DC (average), Horizontal (copy left edge), Vertical (copy top edge). Encode only the residual (difference) to save bits.
4DCT
Discrete Cosine Transform Convert 4×4 spatial blocks to frequency domain. Energy concentrates in top-left (low frequency) coefficients. Similar to JPEG compression — exploits the fact that images have smooth regions.
5Quantize
Quantization Divide DCT coefficients by a quantization matrix and round. High-frequency coefficients (bottom-right) are divided by larger values, often becoming zero. This is the lossy step — controls quality vs size tradeoff.
6Entropy Code
Exp-Golomb VLC Variable-length coding: small values get short codes, large values get longer codes. Zeros (common after quantization) encode very efficiently. Produces the final compressed bitstream.
